
La clave de los datos centrados en el usuario:

- Todas las actividades de user-center-design necesitan input data. 
- Estos datos se procesan para extraer los "User Pain Points".
- Arreglar esos "Pain Points" significa que los usuarios probablemente les encante tu
producto.

- Para lograr esto, primero se define exactamente para quien estas desarrollando y esto se
logra creando "Personas".

- Luego se utilizan técnicas de "ideación" para generar múltiples soluciones para los
"Pain Points" que descubras.

- Luego de la "ideation" se traeran las ideas de vuelta a la realidad al crear "escenarios" 
los cuales luego se utilizaran para construir un prototipo de "user-interface" que contengan
los fixes sugeridos.

- Posteriormente se utilizara "usability test" en la UI para ver si tus ideas funcionan con
la audiencia target. 

- Se crea un plan de desarrollo para crear la interfaz.

- Todo este proceso confía en conseguir datos de buena calidad de usuarios representativos
realizando tareas representativas.
